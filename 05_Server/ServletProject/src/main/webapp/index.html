        1) form태그 : 내부에 작성된 input 값을 지정된 경로로 제출
            - action : 제출할 경로
            - method : 제출 방법(GET/POST)

        2) input 태그 : 값을 입력하는 양식 제공
            - name : 서버로 제출 시 key 역할
            - 입력된 값 또는 value : 서버로 제출 시 value 역할

Get 방식 요청 및 응답

        get 방식 요청
        - 요청하는 주소 뒤에 전달하려는 값을 key=value 형태로 부텽 서버로 전달하는 방식
        http://localhost/example1.do?inputName=홍길동99&inputAge=25
        http://                             -> 프로토콜(규약)
        localhost/example.do                -> 요청 주소
        ?inputName=홍길동&inputAge=25       -> queryString(쿼리스트링)


        [장점]
        - 단순함
        - 주소에 값을 임의로 작성 가능
        - 캐싱(caching, 저장) 가능 : 주소 저장(북마크, 즐겨찾기, 주소공유)

        [단점]
        - (주소창에)글자 수 제한이 있다
        - 보안에 취약(비밀번호가 주소에 보임)

Post 방식 요청 및 응답

        - Get 방식과 다르게 전달 되는 값을 주소가 아닌 HTTP BODY에 담아서 전달하는 방식

        [장점]
        - 길이 제한 없음
        - 데이터를 숨겨서 전달(보안성 향상)

        [단점]
        - 캐싱 불가능(북마크를 하면 원하는 페이지를 볼 수 없음)
        - 서버에서 추가적인 문자 인코딩 처리 필요
